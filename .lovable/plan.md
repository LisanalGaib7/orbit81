## 목표
390px 모바일에서 설정 코그와 모든 서브 항목이 첫 탭에 바로 반응하도록 `HeaderBar`의 터치/포털/외부클릭 구조를 정리합니다.

## 확인한 문제 지점
- `SubIcon`은 모바일에서 Tooltip을 우회하지만, 메인 코그 버튼은 여전히 Radix `Tooltip`으로 감싸져 있어 터치 환경에서 첫 탭을 가로챌 가능성이 남아 있습니다.
- `TemplatePanel`은 서브 메뉴 내부의 절대 위치 패널이라 작은 모바일 화면에서 화면 밖/겹침/외부 클릭 판정 문제가 생길 수 있습니다.
- `profileOpen`은 `closeSubPanels`에서 닫히지 않아, 프로필 모달 상태와 허브 메뉴 상태가 꼬일 수 있습니다.

## 구현 계획
1. **모바일 코그 Tooltip 완전 우회**
   - `SubIcon`과 동일하게 메인 설정 코그도 모바일에서는 `Tooltip` 없이 순수 버튼으로 렌더링합니다.
   - 데스크톱에서는 기존 Tooltip을 유지합니다.

2. **서브 항목 onClick 안정화**
   - Manual / Templates / Pilot Profile 클릭 시 다른 패널 상태를 명확히 닫고, 클릭한 항목의 패널만 열리도록 상태 전환을 정리합니다.
   - `closeSubPanels`에 `profileOpen`도 포함해 메뉴 상태가 남지 않도록 합니다.

3. **모바일 TemplatePanel 포털화**
   - 모바일에서는 템플릿 목록을 `document.body` 포털 기반의 중앙/하단 패널로 띄워 화면 밖으로 밀리지 않게 합니다.
   - 데스크톱에서는 기존처럼 코그 옆 fly-out 형태를 유지합니다.

4. **외부 클릭 판정 개선**
   - 포털 패널이 열려 있을 때 허브의 outside-click 핸들러가 첫 탭을 즉시 닫아버리지 않도록 조건을 정리합니다.
   - 패널 내부 클릭은 닫힘으로 오인되지 않게 유지합니다.

5. **검증**
   - 390px 모바일에서 코그 → Manual → 닫기, 코그 → Templates → 템플릿 선택, 코그 → Pilot Profile(로그인 상태일 때), Reset 2단계, Logout 2단계를 각각 첫 탭 기준으로 확인합니다.
   - 데스크톱 Tooltip 동작이 깨지지 않았는지도 확인합니다.
